## Architecture

This is an Obsidian plugin (TypeScript, esbuild bundled to a single `main.js`). It lets users search selected text on the web via context menu or command palette.

### Source files (`src/`)

- **main.ts** — Plugin entry point. Registers the editor context menu handler and the command palette command. Handles engine selection, URL construction, and settings load/save with preset merging logic.
- **settings.ts** — `PluginSettingTab` implementation. Renders the settings UI with drag-and-drop engine reordering, category management, and three modals (SiteEditModal, AddCategoryModal, RestorePresetsModal). Contains image processing for custom icons (resize to 128×128, data URL conversion).
- **types.ts** — Data models (`SearchEngine`, `WebSearchSettings`, `SiteCategory`), preset engine definitions (`PRESET_ENGINES`), default settings, and `getFaviconUrl` helper.
- **i18n.ts** — Internationalization with English and Japanese translations. Uses `moment.locale()` from Obsidian to detect language. Translation function `t(key, ...args)` with `$1`-style interpolation.

### Key patterns

- **Preset merging**: On load, saved engines are merged with `PRESET_ENGINES` — presets get updated fields from saved data while preserving user customizations. Deleted presets are tracked in `deletedPresetIds` and can be restored.
- **Icon system**: Three icon types — `favicon` (auto-fetched via Google's favicon API), `lucide` (Obsidian built-in icons), `custom` (user-uploaded image stored as data URL). Fallback chain: custom/favicon image → Lucide icon on load error.
- **`MenuItem.dom`**: Obsidian's `MenuItem` has a `dom` property at runtime that isn't in the type definitions. Accessed via `(item as any).dom` to replace menu icons with favicons.

### Build output

- `main.js` — Bundled plugin (gitignored, generated by esbuild)
- `styles.css` — Plugin styles (committed, loaded by Obsidian)
- `manifest.json` — Plugin metadata (required by Obsidian)
